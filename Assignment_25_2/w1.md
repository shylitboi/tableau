## 6강. 데이터결합 혼합

<!-- 데이터결합 및 혼합에 대해 알게 된 점을 적어주세요 -->

* 혼합은 관계/조인과 달리 실제로 결합하지 않음
* SQL의 View 느낌으로 가상 결합
* 2개의 데이터 원본 가져와서 사용
* 당연하지만 하나 이상의 필드(조인키)가 필요함
* 혼합은 굳이 권장하지 않음

## 7강. 데이터 결합과 유니온

<!-- 유니온에 대해 알게 된 점을 적어주세요 -->

* 테이블 구조가 같아야함(필드 수, 유형, 이름이 동일해야함)

## 8강. 라이브와 추출

<!-- 라이브와 추출에 대해 알게 된 점을 적어주세요 -->
* 데이터 원본 페이지에 존재, 원본 페이지는 전처리 하는 공간
* 라이브 : 데이터의 실시간 업데이트, 속도가 떨어짐
* 추출 : 로컬 데이터를 실시간으로 업데이트 하려면 계속 새로고침, 데이터를 다시 올려줘야함

> **🧞‍♀️ 라이브와 추출 방법의 차이가 무엇인가요? 어떤 경우에 추출을 사용하면 좋을까요?**

* 병원에서 일간/주간 추세를 일정 시간마다 모니터링하는 경우..
-> 실시간으로 데이터를 확인할 필요가 없을 때 추출 방식 사용


**참고자료 : 온프레미스 데이터란?**

기업이나 조직이 자체적으로 보유한 물리적 서버나 데이터 센터 내에서 직접 관리하는 데이터로, 조직 내의 하나 이상 서버에서 통제됩니다. 본 서버들은 기업의 데이터센터나 물리적 위치에 설치되어 있으며, 네트워크를 통해 해당 데이터에 접근하고 관리할 수 있습니다.
